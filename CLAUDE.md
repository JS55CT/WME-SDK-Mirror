# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a **documentation mirror and processing system** for the Waze Map Editor (WME) SDK. The project maintains a local copy of the WME SDK TypeDoc documentation and provides a 6-step Python pipeline to:

1. Download fresh HTML from the live Waze SDK docs SPA (Single-Page App)
2. Convert HTML to individual Markdown files
3. Bundle those files for LLM consumption (e.g., docs, Claude context)
4. Keep the WME SDK skill in sync with the latest SDK documentation

**Key difference from traditional web crawlers:** The Waze SDK docs are a TypeDoc SPA — all navigation is generated by JavaScript at runtime. Traditional tools like HTTrack only capture `index.html`. This toolchain instead decodes TypeDoc's embedded navigation bundle (`window.navigationData` in `assets/navigation.js`) to discover all page URLs dynamically.

## Development Workflow

### Prerequisites

One-time setup (Windows):
```powershell
py -m pip install requests beautifulsoup4 lxml
```

This installs the three dependencies needed by all scripts:
- `requests` — HTTP downloads
- `beautifulsoup4` — HTML parsing
- `lxml` — BeautifulSoup XML backend (optional but recommended)

### Running the Full Pipeline

**All scripts must be run from the `production/latest/scripts/` folder as your working directory.**

Full pipeline (copy-paste into PowerShell):
```powershell
cd production/latest/scripts
py cleanup.py; py build-url-list.py; py download-pages.py; py extract-to-md.py; py create-grouped-md-files.py; py update-skill.py
```

Or run them individually:

| Script | Purpose | Output |
|--------|---------|--------|
| `cleanup.py` | Delete old `.md` and `-clean.html` files from output folders | Clears stale output |
| `build-url-list.py` | Fetch live SDK docs SPA, decode TypeDoc navigation bundle, list all page URLs | `url-list.txt`, `url-list-paths.txt` |
| `download-pages.py` | Download HTML pages + fetch latest TypeScript definitions from `https://web-assets.waze.com/wme_sdk_docs/production/latest/wme-sdk-typings.tgz`, extract to `TypeDefs/` | HTML files + `wmeSDK_typedefs.d.ts` |
| `extract-to-md.py` | Parse TypeDoc HTML structure, convert to Markdown with YAML frontmatter | `.md` files alongside HTML |
| `create-grouped-md-files.py` | Bundle all individual `.md` files into topic-grouped outputs (e.g., `classes.md`, `index.md`) | `docs/` folder with LLM-ready bundles |
| `update-skill.py` | Pull latest WME SDK skill from `~/.claude/skills/wme-sdk/SKILL.md` | `skills/SKILL.md` synced with your Claude Code skill |

### Script Options

**`build-url-list.py`:**
```powershell
py build-url-list.py --base-url https://www.waze.com/editor/sdk/  # (default)
py build-url-list.py --out my-urls.txt                            # custom output
```

**`download-pages.py`:**
```powershell
py download-pages.py                    # incremental (skip existing files)
py download-pages.py --force            # re-download everything
py download-pages.py --dry-run          # preview only, no writes
py download-pages.py --delay 0.5        # seconds between requests (default 0.3)
py download-pages.py --no-typedefs      # skip TypeScript definitions from wme-sdk-typings.tgz
```

### Output: docs Folder

After a successful run, the `docs/` folder contains:

- **`index.md`** — Master table of contents with SDK version and source guide
- **Topic bundles:** `classes.md`, `interfaces.md`, `types.md`, `functions.md`, `variables.md`, `modules.md`, `changelog.md`
- **Type definitions:** `wmeSDK_typedefs.d.md`, `geojson_typeddefs.d.md` (wrapped in markdown code blocks)
- **Guides:** `how-to-get-started-with-the-wmeSDK.md`, `migration-guide.md`, etc.
- **External docs:** Turf docs, GeoJSON format spec, geometry converters

Load these files into your LLM context to get complete, up-to-date SDK reference.

### Output: Skills Folder

After a successful run, the `skills/` folder contains:

- **`SKILL.md`** — Copy of the WME SDK skill pulled from `~/.claude/skills/wme-sdk/SKILL.md`

This keeps your skill versioned alongside the SDK documentation it references. The skill is automatically updated whenever you run the full pipeline, ensuring it stays in sync with the latest docs and SDK version.

## Architecture & Design

### The 6-Step Pipeline

1. **Cleanup** → Wipe old output to avoid stale/duplicate content
2. **URL discovery** → Decode TypeDoc's base64+zlib navigation bundle to find all pages
3. **Download** → Fetch fresh HTML + grab latest TypeScript typings tarball
4. **HTML→Markdown** → Parse TypeDoc structure, extract signatures/docs, add YAML frontmatter
5. **Bundle for LLM** → Group individual `.md` files by category, create master index
6. **Sync Skill** → Pull latest WME SDK skill from Claude Code skills directory

### Key Insights

- **SPA crawling** — `build-url-list.py` decodes `window.navigationData` from `assets/navigation.js` rather than crawling HTML links. This is the core innovation that makes the system work.
- **Incremental downloads** — `download-pages.py` defaults to skipping existing files. Use `--force` only when you want to re-fetch everything.
- **YAML frontmatter** — Each `.md` file gets `title`, `source`, `created`, `tool`, `notes` fields. This helps LLMs understand document metadata.
- **Typings sync** — `wmeSDK_typedefs.d.ts` is downloaded from `https://web-assets.waze.com/wme_sdk_docs/production/latest/wme-sdk-typings.tgz`, extracted, and kept in sync with the HTML docs on every refresh (unless `--no-typedefs` is used).

### Git Tracking

The `.git/` directory tracks updates over time. Recent commits show WME version bumps (e.g., v2.345). This lets you see when the SDK was last updated and diff changes between versions.

## Common Tasks

### Update the SDK docs (full refresh)

```powershell
cd latest/
py cleanup.py; py build-url-list.py; py download-pages.py; py extract-to-md.py; py create-grouped-md-files.py; py update-skill.py
git add -A && git commit -m "Updated for WME version v2.XXX"
```

### Check what URLs have changed

```powershell
cd latest/
py build-url-list.py
git diff url-list-paths.txt
```

### Re-download everything (force refresh)

```powershell
cd latest/
py cleanup.py
py download-pages.py --force
py extract-to-md.py
py create-grouped-md-files.py
py update-skill.py
```

### Incremental update (fast, only new/changed pages)

```powershell
cd latest/
py build-url-list.py  # find any new pages
py download-pages.py  # only fetch new/missing files
py extract-to-md.py   # regenerate .md from all HTML
py create-grouped-md-files.py  # rebundle
py update-skill.py  # sync skill
```

### Update only the skill (without re-fetching SDK docs)

If you've updated your skill in `~/.claude/skills/wme-sdk/SKILL.md` but the SDK itself hasn't changed:

```powershell
cd latest/
py update-skill.py
git add skills/SKILL.md && git commit -m "Updated WME SDK skill"
```

## Skill Integration & Documentation Fallback Chain

### How `update-skill.py` Works with Fallback Chain

`update-skill.py` pulls your WME SDK skill from your Claude Code skills directory and keeps a copy in the `skills/` folder. It also **injects a fallback chain** so the skill can reference documentation in two ways:

1. **Version control** — Skill is tracked in git alongside the SDK version it documents
2. **Offline + online support** — Skill references local docs first, GitHub Pages as fallback
3. **Workflow integration** — Skill updates automatically when you run the full pipeline

The script:

- Reads from `~/.claude/skills/wme-sdk/SKILL.md` (default)
- Replaces hardcoded paths with a **fallback chain**:
  - **Primary:** `../../docs/` (local, from `skills/` → repo root, works if repo is cloned)
  - **Fallback:** `https://js55ct.github.io/WME-SDK-Mirror/docs/` (GitHub Pages, works anywhere with internet)
- Writes to `skills/SKILL.md`
- Compares content before writing (skips if unchanged)
- Reports status: `[OK]` if current, `[UPDATED]` if new version pulled, `[ERROR]` if source not found

This means:

- **With repo cloned locally** → Skill uses local `../../docs/` paths (offline-friendly)
- **Without repo cloned** → Skill references GitHub Pages URLs for documentation

**Options:**

```powershell
py update-skill.py
py update-skill.py --skill-path C:\custom\path\to\SKILL.md
```

If the skill is not found, verify it exists at `~/.claude/skills/wme-sdk/SKILL.md`. The skill and its documentation stay in sync automatically.

## Technical Notes

### Python Scripts Don't Import Each Other

Each script is standalone — they communicate via files on disk (`url-list.txt`, HTML files, `.md` files). This keeps them modular and easy to run individually or in any order (though the prescribed order is required for a clean result).

### Working Directory Requirement

**Always run scripts from `production/latest/scripts/`.** The scripts use `os.getcwd()` to determine where to read/write files. Running from a different directory will cause path resolution errors.

### Proxy & SSL

`build-url-list.py` and `download-pages.py` include corporate proxy support:
- `requests.Session()` with a Mozilla User-Agent (to avoid blocks)
- `urllib3.disable_warnings()` to suppress InsecureRequestWarning (corporate SSL inspection)
- `verify=False` in requests (skips certificate verification for proxied traffic)

If these cause issues in your environment, remove or adjust them.

### BeautifulSoup Parser

`extract-to-md.py` uses BeautifulSoup with `lxml` for HTML parsing. If `lxml` is not installed, it falls back to `html.parser` (built-in but slower).
